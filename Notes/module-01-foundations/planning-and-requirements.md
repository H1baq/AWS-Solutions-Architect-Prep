# Planning and Requirements in AWS Architecture

## Planning

Planning is not only the initial step in architecting, but a critical step throughout the lifecycle of a solution.

A planning framework provides a structured approach to architecting while ensuring all critical aspects are considered.

---

## Key Planning Principles

- Start with business requirements
- Design for failure
- Implement security at every layer
- Automate where possible
- Use managed services
- Monitor and measure
- Stay current

---

## AWS Architecture Resources

The AWS Architecture Center provides a comprehensive collection of resources to help build well-architected systems and applications.

Offerings include:
- Reference architectures
- Documentation
- Ready-to-deploy solutions

---

## Functional vs Non-Functional Requirements

### Functional Requirements

Functional requirements define what a product or system must do, including its features and functions.

- Primary way customers communicate requirements
- Help keep project teams aligned
- Define system behavior and capabilities
